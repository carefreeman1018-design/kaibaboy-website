## 二、部署（建議：GitHub Pages，免費）

GoDaddy 目前掛的是「即將推出」佔位頁——**Apple 明文不接受佔位頁**，所以一定要換成這個網站。

### 步驟 A：推上 GitHub（跟 Claude 說一聲即可代勞）
```
gh repo create kaibaboy-website --public --source . --push
gh api -X POST repos/carefreeman1018-design/kaibaboy-website/pages -f "source[branch]=main" -f "source[path]=/"
```
（`CNAME` 檔已備好，內容為 kaibaboy.com.tw，GitHub Pages 會自動綁定自訂網域）

### 步驟 B：到 GoDaddy 改 DNS（需要你本人登入操作）
GoDaddy → 我的產品 → 網域 kaibaboy.com.tw → DNS 管理：

1. **刪除**現有兩筆 `A` 記錄（13.248.243.5、76.223.105.230，這是舊的建站工具）
2. **新增四筆 A 記錄**，名稱都是 `@`：
   - `185.199.108.153`
   - `185.199.109.153`
   - `185.199.110.153`
   - `185.199.111.153`
3. 把 `www` 的 CNAME 改指向 `carefreeman1018-design.github.io`
4. ⚠️ **絕對不要動 MX 記錄**（aspmx.l.google.com 等五筆）——那是 dev@kaibaboy.com.tw 的 Google Workspace 收信設定，刪了會收不到信（包括 Apple 和鄧白氏的審核信！）

### 步驟 C：開啟 HTTPS
DNS 生效後（十分鐘到數小時），到 GitHub repo → Settings → Pages → 勾選 **Enforce HTTPS**（憑證需等 GitHub 自動簽發，最多一天）。

## 三、上線後「必補」事項（Apple 審核關鍵）

Apple／鄧白氏會比對官網、D-U-N-S 資料庫、公司登記三方資訊是否一致。**這是台灣公司最常卡關的地方。** 拿到公司登記後，把以下資料告訴 Claude 即可全站更新：

- [ ] **公司法定全名**（中文，例如「海馬博一有限公司」——Apple 不接受行號/商號，必須是法人）
- [ ] **公司英文名稱**（要與鄧白氏登記、Apple 註冊表單一字不差）
- [ ] **統一編號**
- [ ] **登記地址**（中英文，與鄧白氏一致）
- [ ] **聯絡電話**（可實際接聽）

要補進的位置：全站頁尾、`/support/` 聯絡我們清單、`/hippo-eyes/privacy/` 第十節、首頁聯絡區塊、首頁 JSON-LD（legalName／taxID／address／telephone）。

## 四、Apple Developer 申請流程建議順序

1. **網站先上線**（本指南步驟二）→ 確認 https://kaibaboy.com.tw 可正常開啟
2. **查詢／申請 D-U-N-S 編號**：developer.apple.com/enroll/duns-lookup/（免費；台灣實務約 7–14 個工作天）
   - 申請時官網欄位填 `https://kaibaboy.com.tw`，聯絡信箱用 dev@kaibaboy.com.tw
   - 公司名稱、地址、電話務必與官網顯示一致
3. **申請 Apple Developer Program 組織帳號**（年費 USD 99）
   - 用 dev@kaibaboy.com.tw 建立 Apple ID 並開啟雙重認證，姓名填法定姓名（護照拼音）
   - 送出後 Apple 通常 3–5 個工作天內來電核實，準備好 Enrollment ID、D-U-N-S 編號與公司登記文件
   - 申請人若非負責人，需高階主管授權

## 五、河馬眼睛送審（App Review）前檢查清單

- [ ] App Store Connect 三個網址欄位填：Support URL＝`https://kaibaboy.com.tw/support/`、Privacy Policy URL＝`https://kaibaboy.com.tw/hippo-eyes/privacy/`、Marketing URL＝`https://kaibaboy.com.tw/hippo-eyes/`
- [ ] **確定影像處理架構後修改隱私權政策第三、五節**：純裝置端辨識→改寫成「僅於裝置端處理、不上傳伺服器」；有伺服器辨識→明確揭露服務名稱，並與 App Privacy 問卷（營養標籤）答案一致，否則有 5.1.1/5.1.2 退件風險
- [ ] 隱私權政策連結也要放在 **App 內**（例如設定頁），這是審核指南 5.1.1 的硬性要求
- [ ] 把全站「開發中」字樣更新（hippo-eyes 頁徽章與 CTA、首頁徽章、支援中心 FAQ「什麼時候可以下載」），改為「已上架」＋App Store 下載連結
- [ ] 衛福部紅黃綠標示指引正式公告後，同步更新 `/hippo-eyes/` 分級門檻數值
- [ ] 隱私權政策撰寫時的技術假設（不需帳號、掃描紀錄僅存裝置端、無第三方 SDK、非 13 歲以下兒童為對象），若與實際 App 不符要先改政策

## 六、日常維護

- 改完檔案後：`git add -A && git commit -m "說明" && git push`，GitHub Pages 約 1 分鐘自動更新
- `/support/` 與 `/hippo-eyes/privacy/` 網址**永遠不要失效或改路徑**（已填入 App Store Connect 的網址壞掉會被退件）
- 內容有大改時記得更新 `sitemap.xml` 的 `lastmod`
